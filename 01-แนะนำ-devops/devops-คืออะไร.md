# บทที่ 1 - แนะนำ DevOps

> **DevOps** คือแนวทางการทำงานร่วมกันระหว่างทีมพัฒนา (Development: Dev) และทีมปฏิบัติการระบบ (Operations: Ops) เพื่อเพิ่มความรวดเร็ว คุณภาพ และความเสถียรในการพัฒนาซอฟต์แวร์ โดยเน้นการทำงานแบบอัตโนมัติ การทดสอบอย่างต่อเนื่อง และการปล่อยระบบอย่างสม่ำเสมอ บทนี้จะอธิบายแนวคิดพื้นฐานของ DevOps รวมถึงเครื่องมือที่ใช้ในระบบ DevOps ตลอดทั้งโครงงานนี้

#### **เครื่องมือที่ใช้ในคู่มือนี้**

* GitLab (Source Code Management และ CI/CD)
* Docker และ Docker Compose
* Proxmox VE และ Virtual Machine
* Ubuntu Server
* Node-RED

#### **แนวคิดสำคัญของ DevOps**

* การทำงานแบบอัตโนมัติ (Automation)
* การพัฒนา ทดสอบ และปล่อยงานอย่างต่อเนื่อง (Continuous Integration / Continuous Delivery)
* การสังเกตและติดตามระบบ (Observability)
