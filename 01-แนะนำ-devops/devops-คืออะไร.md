# บทที่ 1 - แนะนำ DevOps

**DevOps** คือแนวทางการทำงานร่วมกันระหว่างทีมพัฒนา (Dev) และปฏิบัติการ (Ops) เพื่อส่งมอบซอฟต์แวร์ได้ไว มีคุณภาพ และเชื่อถือได้ โดยเน้น:

* การทำงานแบบอัตโนมัติ (Automation)
* การทดสอบและปล่อยงานบ่อย (Continuous Integration/Delivery)
* การสังเกตการณ์ระบบ (Observability)

**เครื่องมือในคู่มือนี้**

* GitLab (SCM + CI/CD)
* Docker & Docker Compose
